# Quickstart

This guide will get you up and running with a working client-server application in just a few minutes. We'll build a simple echo server that receives messages from clients and sends responses back.

## Core Concepts

Before we start coding, here are the key concepts:

<CardGroup cols={2}>
  <Card title="Server Side" icon="server">
    - Subclass `Server` to create your game server
    - Subclass `Channel` to represent each connected client
    - Call `server.Pump()` once per game loop
  </Card>
  
  <Card title="Client Side" icon="user">
    - Use the singleton `connection` object to connect
    - Subclass `ConnectionListener` to receive events
    - Call `connection.Pump()` and `listener.Pump()` per loop
  </Card>
</CardGroup>

## Building the Server

Let's create a simple server that accepts connections and responds to messages.

<Steps>
  <Step title="Create a Channel class">
    The `Channel` class represents a single connected client on the server side. Each time a client connects, a new instance is created.
    
    ```python server.py
    from time import sleep
    from PodSixNet.Server import Server
    from PodSixNet.Channel import Channel
    
    class ClientChannel(Channel):
        """
        This is the server representation of a single connected client.
        """
        def Network(self, data):
            print("Received from client:", data)
        
        def Network_message(self, data):
            # This is called when action="message"
            print("Client message:", data['message'])
            # Send a response back to this client
            self.Send({"action": "response", "message": "Server received: " + data['message']})
    ```
    
    <Info>
    The `Network()` method is called for all incoming data. The `Network_message()` method is only called when the data contains `{"action": "message"}`.
    </Info>
  </Step>
  
  <Step title="Create a Server class">
    The `Server` class manages all client connections and the main server loop.
    
    ```python server.py
    class MyServer(Server):
        channelClass = ClientChannel
        
        def Connected(self, channel, addr):
            print('New connection from:', addr)
            # Optionally send a welcome message
            channel.Send({"action": "welcome", "message": "Welcome to the server!"})
    ```
    
    <Note>
    The `channelClass` attribute tells the server which Channel class to instantiate for each new connection.
    </Note>
  </Step>
  
  <Step title="Launch the server">
    Create a server instance and run the main loop:
    
    ```python server.py
    if __name__ == '__main__':
        # Start server on localhost:31425
        myserver = MyServer(localaddr=('localhost', 31425))
        print("Server started on localhost:31425")
        
        while True:
            myserver.Pump()
            sleep(0.0001)
    ```
    
    <Tip>
    The `sleep(0.0001)` prevents the loop from consuming 100% CPU while still being responsive.
    </Tip>
  </Step>
</Steps>

## Building the Client

Now let's create a client that connects to the server and sends messages.

<Steps>
  <Step title="Create a ConnectionListener class">
    The `ConnectionListener` class receives network events from the server.
    
    ```python client.py
    from time import sleep
    from PodSixNet.Connection import connection, ConnectionListener
    
    class MyClient(ConnectionListener):
        def __init__(self, host, port):
            self.Connect((host, port))
            print("Client started")
        
        def Network_connected(self, data):
            print("Connected to server")
            # Send a message once connected
            connection.Send({"action": "message", "message": "Hello from client!"})
        
        def Network_welcome(self, data):
            print("Server welcome:", data['message'])
        
        def Network_response(self, data):
            print("Server response:", data['message'])
        
        def Network_error(self, data):
            print('Network error:', data['error'][1])
            connection.Close()
        
        def Network_disconnected(self, data):
            print('Disconnected from server')
    ```
    
    <Info>
    The built-in events `connected`, `disconnected`, and `error` are automatically sent by PodSixNet.
    </Info>
  </Step>
  
  <Step title="Connect and run the client loop">
    Create a client instance and run the main loop:
    
    ```python client.py
    if __name__ == '__main__':
        client = MyClient('localhost', 31425)
        
        while True:
            connection.Pump()
            client.Pump()
            sleep(0.001)
    ```
    
    <Warning>
    You must call both `connection.Pump()` and `client.Pump()` in your game loop. The connection pump handles socket I/O, while the client pump dispatches events to your Network_* methods.
    </Warning>
  </Step>
</Steps>

## Running Your First Server and Client

<Steps>
  <Step title="Start the server">
    In one terminal, run:
    
    ```bash
    python server.py
    ```
    
    You should see:
    ```
    Server started on localhost:31425
    ```
  </Step>
  
  <Step title="Start the client">
    In another terminal, run:
    
    ```bash
    python client.py
    ```
    
    You should see:
    ```
    Client started
    Connected to server
    Server welcome: Welcome to the server!
    Server response: Server received: Hello from client!
    ```
  </Step>
</Steps>

## Complete Working Example

Here's a complete, minimal example based on the real PodSixNet source:

<CodeGroup>
```python Simple Server
from time import sleep
from PodSixNet.Server import Server
from PodSixNet.Channel import Channel

class ClientChannel(Channel):
    def Network_message(self, data):
        print("Message from client:", data['message'])
        self.Send({"action": "echo", "message": data['message']})

class EchoServer(Server):
    channelClass = ClientChannel
    
    def Connected(self, channel, addr):
        print('Client connected:', addr)

if __name__ == '__main__':
    server = EchoServer(localaddr=('localhost', 8000))
    print('Echo server started on port 8000')
    while True:
        server.Pump()
        sleep(0.0001)
```

```python Simple Client
from time import sleep
from PodSixNet.Connection import connection, ConnectionListener

class EchoClient(ConnectionListener):
    def Network_connected(self, data):
        print("Connected!")
        connection.Send({"action": "message", "message": "Hello World"})
    
    def Network_echo(self, data):
        print("Server echoed:", data['message'])
    
    def Network_error(self, data):
        print('Error:', data['error'][1])

if __name__ == '__main__':
    client = EchoClient()
    connection.Connect(('localhost', 8000))
    
    while True:
        connection.Pump()
        client.Pump()
        sleep(0.001)
```
</CodeGroup>

## How the Action System Works

PodSixNet uses an action-based routing system:

```python
# When you send this:
connection.Send({"action": "jump", "height": 10, "player_id": 5})

# PodSixNet looks for this method:
def Network_jump(self, data):
    # data = {"action": "jump", "height": 10, "player_id": 5}
    player_id = data['player_id']
    height = data['height']
```

<Info>
If no `Network_<action>` method exists, the fallback `Network(data)` method is called instead (if it exists).
</Info>

## Sending Different Data Types

PodSixNet can serialize Python data structures automatically:

```python
# Dictionaries
connection.Send({"action": "update", "position": {"x": 100, "y": 200}})

# Lists
connection.Send({"action": "players", "list": ["Alice", "Bob", "Charlie"]})

# Mixed types
connection.Send({
    "action": "gamestate",
    "players": [{"name": "Alice", "score": 100}, {"name": "Bob", "score": 85}],
    "time_remaining": 42.5,
    "level": 3
})
```
